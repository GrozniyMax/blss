# Доменная модель ЛР

### 1. DeliveryPoint (Точка доставки)
Представляет физическое местоположение, где можно получить или получить заказы.

**Поля:**
- `id`: UUID
- `name`: String (Не может быть NULL)
- `address`: String (Не может быть NULL)

Ограничения:
- совокупность name и address должна быть уникальной

### 2. Product (Продукт)
Представляет продукт, доступный в системе.

**Поля:**
- `id`: UUID
- `name`: String (Не может быть NULL, уникальное значение)
- `price`: BigDecimal (Не может быть NULL)

### 3. User (Пользователь)
Представляет клиента или пользователя системы.

**Поля:**
- `id`: UUID
- `email`: String (Не может быть NULL, уникальное значение)

### 4. Order (Заказ)
Представляет заказ клиента.

**Поля:**
- `id`: UUID
- `status`: Status (Не может быть NULL)
- `owner`: UUID (Не может быть NULL)
- `localtion`: UUID (Не может быть NULL)
- `totalAmount`: BigDecimal (Не может быть NULL)
- `creationDate`: Instant (Не может быть NULL)
- `lastEdited`: Instant (Может быть NULL)

### 5. OrderItem (Элемент заказа)
Представляет элемент внутри заказа.

**Поля:**
- `id`: UUID
- `orderId`: UUID (Не может быть NULL)
- `productId`: UUID (Не может быть NULL)
- `yacheyka`: String (Может быть NULL)

### 6. StoreItem (Элемент склада)
Представляет информацию о запасах продукта.

**Поля:**
- `productId`: UUID
- `count`: Integer (Положительное значение)

### 7. Status (Статус)
Перечисление, представляющее возможные состояния заказа.

**Значения:**
- `CREATED`: Заказ создан
- `PROCESSING`: Заказ обрабатывается
- `IN_DELIVERY`: Заказ доставляется
- `READY_FOR_PICKUP`: Заказ готов к выдаче
- `CANCELED`: Заказ отменен или возвращен
- `DONE`: Клиент получил заказ
